Installation

Learn how to deploy Jellyplot, it's fast and easy!

Clone GitHub Repository

git clone https://github.com/ldvloper/jellyplot.git

Configure your Environment File

Change the variables in the .env file to apply your settings, customize the planner with your corporate/brand colors and turn on/off functionalities.

Configurate your environment variables

Copy and change the environment file and set your configuration $~ cp .env.example .env

Install vendor folder

Execute the following code docker run --rm \ -u "$(id -u):$(id -g)" \ -v $(pwd):/opt \ -w /opt \ laravelsail/php80-composer:latest \ composer install --ignore-platform-reqs

Run Docker and execute Laravel Sail

Run the following commands to run Laravel Sail $~ ./vendor/bin/sail up

Generate a new Application Key

Generate a fresh application key for encryption $~ sail php artisan key:generate

Run the Database Migration

Create a Database and execute the artisan commands $~ sail php artisan migrate
$~ sail php artisan db:seed

Install The Front End Packages

Install NodeJS Dependencies and run the following commands $~ sail npm install
$~ sail npm run dev/prod

Content

JellyPlot is a lightweight planning tool based on PHP Laravel framework and VueJS. Which allows you to organize various projects and tasks in an organized and simple way.

With JellyPlot you will be able to have a robust planning system you also can create and manage API's with a intuitive interface and more.

When we started to develop JellyPlot we always focused on offering the best facilities and simplicity at the time of deployment.

Structure

  • Tasks: That will be the information displayed in the scheduler.
  • Projects: Projects are associated with tasks and customers.
  • Resources: The Resources/Rooms/Places are associated with tasks.
    Each Task takes place at a given time lapse and on a given resource.
  • Equipment: These are the equipment you will work with, it will remain reserved for the duration of the task to which you have assigned them.
  • Customers: These are your clients, you will be able to associate indicators for a good data analysis.

When you install Jellyplot by default you will find that there are already clients, projects, users and equipment created.

You can do tests with this data to understand the Planning Tool Structure.

Admin Credentials

Once the database seed is done, you will be able to access the content with the following credentials:

  • Email: admin@jellyplot.com
  • Password: admin

You can modify these credentials in the future changing your .env file.
